{% sectionCopy eyebrow="Step One" title="Set the Site Identity Once." %}
One configuration surface controls the site name, canonical URL, navigation, footer copy, and default metadata. Rebranding or launch prep should not require hunting through route files.

That keeps the system understandable before anyone writes a page.
{% /sectionCopy %}

{% sectionCopy eyebrow="Step Two" title="Write Content Inside Approved Blocks." tone="subtle" %}
New pages and revisions are written as markdown that calls prebuilt blocks. That keeps writing focused on content instead of reopening layout, styling, or template decisions every time.

Humans and agents use the same publishing surface, so the workflow does not fork as the team grows.
{% /sectionCopy %}

{% sectionCopy eyebrow="Step Three" title="Validate Before You Publish." %}
A content check step catches route collisions, bad frontmatter, and invalid block usage before a change becomes accepted content. The readable source stays in the repo, and the runtime ships from generated state built from accepted pages.

That is how PageQuarry keeps editing fast without making the site brittle.
{% /sectionCopy %}
