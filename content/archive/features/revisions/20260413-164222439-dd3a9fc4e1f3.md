{% sectionCopy eyebrow="Presentation Layer" title="Blocks, Templates, and Site Chrome Stay Stable." %}
Design decisions live in React. Blocks, templates, navigation, footer copy, and metadata defaults all stay in code where developers can review and evolve them deliberately.

That means a fast content change does not have to reopen the visual system.

{% linkItem href="/how-it-works" label="How It Works" summary="See how the publishing model moves content from a draft to the live site." /%}
{% /sectionCopy %}

{% sectionCopy eyebrow="Publishing Surface" title="Humans and Agents Write Inside Approved Blocks." tone="subtle" %}
Writers and agents publish through structured markdown. They fill approved inputs, reuse prebuilt presentation, and stay inside the same design boundaries every time.

That is what makes AI-assisted publishing usable on a real site instead of turning every edit into layout roulette.

{% linkItem href="/howto/editorial/publishing-workflow" label="Publishing Workflow" summary="Read the step-by-step guide for the content flow." /%}
{% /sectionCopy %}

{% sectionCopy eyebrow="Operations" title="Validation, History, and Deploys Stay Predictable." %}
PageQuarry validates drafts before they are accepted, keeps readable source in the repo, and builds the runtime from trusted content instead of raw working files.

The result is a publishing model that stays legible under real team use, including messy AI-assisted editing sessions.
