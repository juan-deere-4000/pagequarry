{% sectionCopy eyebrow="Problem" title="The Site Needed to Explain the Product and Prove It." %}
A product about design-safe publishing should not rely on hidden admin tools or one-off page code for its own marketing site. The proof has to be visible in the workflow itself.

That meant the public site had to run on the same content model the product is selling.
{% /sectionCopy %}

{% sectionCopy eyebrow="Build" title="The Solution Was to Put the Marketing Site on the Product Model." tone="subtle" %}
Blocks, templates, navigation, footer copy, and metadata live in code. Homepage copy and supporting pages publish through markdown. That gives the site a stable design system with a fast writing surface.

The site can change quickly without becoming a separate stack that only proves the marketing team can ship pages.
{% /sectionCopy %}

{% sectionCopy eyebrow="Result" title="Content Can Move Fast Without Reopening Design Decisions." %}
New copy, revisions, and supporting pages can ship through the same structured path the product promises. That makes the site a concrete example instead of a detached brochure.
