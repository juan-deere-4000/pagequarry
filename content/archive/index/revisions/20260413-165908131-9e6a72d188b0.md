{% sectionCopy eyebrow="Who It Is For" title="For Teams Shipping React Sites with AI-Assisted Publishing." %}
PageQuarry is for teams that want AI help with writing and editing, but do not want every prompt to become a new design decision. It fits product sites, docs, editorial hubs, and marketing pages that already live in a React codebase.

You keep the visual system in code. Humans and agents publish inside approved blocks instead of improvising layouts, styling, or SEO rules from scratch.
{% /sectionCopy %}

{% sectionCopy eyebrow="What Changes" title="Your Agent Can Write the Page Without Redesigning It." tone="subtle" %}
With a normal CMS, content velocity often comes with design drift. With PageQuarry, the writing surface stays small and the design system stays stable.

- Build blocks once in React
- Publish new content through markdown calls
- Keep templates, headers, footers, and metadata stable
- Review accepted source directly in the repo
{% /sectionCopy %}

{% sectionCopy eyebrow="Why It Matters" title="You Get Speed Without Handing the Site Over to a Generator." %}
Teams adopt PageQuarry when they want AI help with publishing but still need the site to look like it was designed on purpose. Developers keep ownership of the system. Writers and agents work inside it.

That means fewer visual regressions, fewer layout debates, and a clearer path from draft to production.
{% /sectionCopy %}

{% sectionCopy eyebrow="React Example" title="Define the Presentation Once." %}
A developer creates a block with the spacing, typography, and CTA treatment the site should keep using. After that, every page can reuse the same presentation instead of reinventing it.

That is what keeps fast content work from turning into front-end churn.
{% /sectionCopy %}

{% sectionCopy eyebrow="Markdown Example" title="Publish New Pages Without Opening the Layout." tone="subtle" %}
Humans and agents call that block from markdown by filling approved inputs. New content moves faster because the writing surface is small and the design system stays intact.

That is the core promise of PageQuarry: faster publishing without giving up control of the presentation layer.
{% /sectionCopy %}

{% sectionCopy eyebrow="Supporting System" title="Templates, Navigation, SEO, and Validation Stay in Code." %}
PageQuarry keeps the surrounding system in code: templates, site chrome, metadata defaults, redirects, and validation. That supporting infrastructure is what lets the publishing surface stay simple without making the site fragile.

- Templates define what each page type can publish
- Site chrome and metadata stay consistent
- Accepted content stays visible in the repo
- Validation catches bad edits before they ship
{% /sectionCopy %}
