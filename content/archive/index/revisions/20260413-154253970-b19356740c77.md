{% sectionCopy eyebrow="What It Is" title="Build the design system once, then let your agent publish inside it." %}
PageQuarry separates the design system from the writing surface. You build your site in React, create reusable blocks and templates, and keep control of layout, styling, metadata, and site structure.

Then you or your agent write content in markdown using those predefined blocks. Content work gets faster, while the design stays consistent.

{% linkItem href="https://github.com/juan-deere-4000/pagequarry" label="View on GitHub" summary="See the public repo and current framework." /%}
{% /sectionCopy %}

{% sectionCopy eyebrow="Why It Works" title="Agents can write content. They should not have to redesign the page." tone="subtle" %}
Coding agents are good at writing, revising, and organizing content. They are much less trustworthy when every edit also invites layout drift, styling churn, or visual improvisation.

PageQuarry gives them a constrained surface. They can fill in a system without redesigning it.
{% /sectionCopy %}
