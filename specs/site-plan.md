# PageQuarry Site Plan

## Goal
Ship a simple one-page site for PageQuarry at pagequarry.com.

## Constraints
- One page only
- Primary CTA goes to the public GitHub repo
- Use the CMS as the real publishing path
- Beautiful on desktop and mobile
- Use Playwright during design and refinement
- Track bugs and rough edges as GitHub issues in the CMS repo

## Working Shape
- Hero
- Tight explanation of what PageQuarry is
- Why it exists / core principles
- How it works in a few concrete steps
- GitHub CTA

## Design Direction
- Calm, editorial, minimal
- Strong typography
- Warm light background
- Sparse accent color
- Feels like a product, not an agency page

## Execution Notes
- Keep site-specific changes in site-owned seams when possible
- Use markdown docs in specs/ to track decisions and rough edges
- Validate repeatedly with lint, test, build, and content audit
