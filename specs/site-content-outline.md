# PageQuarry Site Content Outline

## Hero
Eyebrow: markdown-first sites
Title: a calm publishing system for people who want to own their website.
Deck: PageQuarry is a one-repo site framework with structured markdown publishing, generated runtime state, and enough opinion to keep future edits sane.
Aside: no dashboard CMS drift, no mystery admin state, no fragile runtime reading whatever file happened to exist.
CTA: go to GitHub

## Section 1
What it is.
A forkable, self-hostable site system. Clear seams. Real files. Predictable publishing.

## Section 2
Why it feels different.
The point is not infinite flexibility. The point is staying coherent after the tenth edit, the hundredth commit, and the first agent you let touch production.

## Section 3
How it works.
Configure site identity. Write or revise markdown. Publish through validation. Ship from generated state.

## Section 4
Principles.
Visible content history. Narrow editing surface. Recoverable mistakes. Static output.
